# create-node Skill

## 工作流程概览

续写任务分四个阶段依次执行：

1. **信息收集** — 读取规划文件、已有内容、设定文档
2. **委托撰写** — 向 Agent 传入完整上下文和参数
3. **质量检查** — 自动验证字数、一致性、连贯性、文风
4. **修复补齐** — 对不达标的部分自动扩充或重写

---

## 阶段一：信息收集

### 1.1 定位输入文件

按优先级自动搜索以下文件：

**规划/大纲文件：**
- `剧情规划.txt`、`outline.txt`、`大纲.txt`、`plan.txt`
- `小说剧情规划.txt`、`story_outline.md`
- 用户指定的任何规划文件

**已有内容文件：**
- `小说_X-Y章.txt`、`chapter_X-Y.txt`
- `story_part1.txt`、`part_a.md`
- 按文件名排序，找到最新的已写章节

**设定文档：**
- `人物设定.txt`、`setting.md`、`世界观.txt`
- `characters.md`、`worldbuilding.txt`

如果找不到规划文件，从已有章节内容中反向推导剧情走向。

### 1.2 读取并提取关键信息

读取以下内容（至少读最近一章和规划文件）：

**从规划文件中提取：**
- 续写范围的章节编号和标题
- 每章的剧情要点和关键事件
- 出场人物
- 境界/进度设定

**从已有内容中提取：**
- 主角姓名、性格特征（冷静/果断/霸道/护短等）
- 配角姓名和关系
- 文风特征（段落长度、对话密度、形容词使用频率、比喻风格）
- 当前剧情进度和未解决的伏笔
- 修为/等级/世界观设定
- 章节分隔符格式（如 `----------------------`）

**从设定文档中提取：**
- 人物关系图谱
- 世界观规则
- 专有名词列表

### 1.3 向 Agent 传递的上下文

构造 Agent prompt 时，必须包含：

```
1. 剧情规划：续写范围内每章的大纲要点
2. 前情提要：最后一章的关键情节摘要（至少读最后500字）
3. 人物设定：主角性格锚点（不可更改）、配角关系、出场人物清单
4. 文风要求：段落长度、对话风格、比喻偏好、分隔符格式
5. 硬约束：每章最少字数、总章节数、输出文件名
6. 剧情连贯性要求：不能出现人物突然死亡又复活、修为倒退、名字写错等问题
```

---

## 阶段二：委托撰写

### 2.1 启动 Agent

使用 `Agent` 工具，传入构造好的 prompt。关键参数：

- `description`: 简短描述任务（如 "Write chapters X-Y"）
- `prompt`: 完整的上下文 + 每章大纲 + 文风要求 + 硬约束
- `run_in_background`: `true`（后台运行，不阻塞）

### 2.2 命名规范

- 输出文件名格式：`{基础名}_{起始}-{结束}{扩展名}`
- 如果目标文件已存在且为空，直接覆盖
- 如果目标文件已有内容，追加到末尾（需确认）

---

## 阶段三：质量检查

撰写完成后，立即执行以下检查。每项检查不通过则进入阶段四修复。

### 3.1 字数达标检查

按章节统计中文字符数（去除空白字符）：

```powershell
# 示例：按分隔符拆分章节并统计
$content = Get-Content "输出文件路径" -Encoding UTF8 | Out-String
$chapters = $content -split '----------------------'
foreach($i in 0..($chapters.Count-1)) {
    $text = $chapters[$i].Trim()
    if($text.Length -gt 10) {
        $chars = ($text -replace '\s','').Length
        # 如果 $chars < 最小字数要求，标记为 SHORT
    }
}
```

- 网文章节默认不少于 1500 字（可在 prompt 中配置）
- 技术文档段落默认不少于 300 字

### 3.2 人物一致性检查

检查续写内容中的人物设定是否与已有内容一致：

- **主角姓名**不能前后不一致（如前面叫"林墨"，后面不能变成"林辰"）
- **性格特征**不能突变（如前面冷静果断，后面不能变得优柔寡断）
- **修为/等级**不能倒退（除非剧情有明确突破描写）
- **角色关系**不能错乱（如"楚清瑶"是圣女身份，不能突然变成侍女）
- **出场人物**要符合规划（不该出场的人不要突然出现）

实现方式：从已有内容中提取人物特征清单，在续写内容中 grep 验证。

### 3.3 剧情连贯性检查

- 续写的第一章必须与上一章的**最后一段**自然衔接
- 不能有"遗忘"前文关键情节的情况（如前文刚杀了某人，后文装作没发生过）
- 时间线要合理（不能前一天说"三日后"，后一天就发生了）
- 伏笔要呼应（前文埋的线索要在后续有交代）

### 3.4 文风匹配检查

对比续写内容和已有内容的：

- **段落长度**：是否都是短句短段（网文典型特征）还是突然变成长段落
- **对话密度**：对话占比是否合理
- **形容词使用**：是否过度使用或完全不用
- **比喻风格**：是否有类似的比喻手法
- **叙述节奏**：是否保持了类似的快慢节奏

如果文风差异明显，需要在阶段四重写受影响的部分。

---

## 阶段四：修复补齐

### 4.1 字数不足 → 扩充内容

对字数不达标的章节，在**章节内部**追加内容，而不是简单拉长句子。扩充方向：

- 补充战斗过程的细节描写
- 增加人物对话和内心活动
- 扩展环境氛围描写
- 添加后续影响的交代

### 4.2 人物不一致 → 修正

- 人名错误：全局替换为正确名字
- 性格偏差：重写相关段落，使其符合人物锚点
- 修为错误：修正数字和描述

### 4.3 文风不匹配 → 重写

如果某章文风与整体差异过大，重写该章，遵循以下原则：

- 使用短段落（每段 2-5 行）
- 对话精简，少用修饰性标签（如"愤怒地说道"改为直接对话+动作）
- 动作导向，少静态描写
- 比喻要锋利具体，避免空洞的形容词堆砌

### 4.4 二次验证

修复完成后，重新执行阶段三的全部检查，确保所有指标达标。

## 配置参数

用户可通过以下方式自定义：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `start_chapter` | 自动检测最新章节+1 | 起始章节编号 |
| `end_chapter` | start + 9 | 结束章节编号 |
| `min_words_per_chapter` | 1500 | 每章最少字数 |
| `output_file` | 自动推断 | 输出文件路径 |
| `encoding` | UTF8 | 文件编码 |
| `separator` | `----------------------` | 章节分隔符 |

---

## 注意事项

1. **永远不要改变主角性格** — 这是最高优先级规则
2. **不要跳过前文** — 必须读完最后一章的最后500字再开始写
3. **不要自行决定剧情走向** — 严格按照规划文件执行，没有规划时才从已有内容推导
4. **发现问题及时汇报** — 如果规划文件和已有内容存在矛盾（如规划说主角叫"林辰"但正文写的是"林墨"），先指出让用户确认
5. **文件编码** — Windows 环境下注意使用 UTF8 编码读写，避免乱码
